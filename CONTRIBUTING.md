## How to Contribute
1. **Report bugs**: Use the GitHub issues tracker to report any bugs you find.
2. **Suggest features**: Have an idea? Open an issue to discuss it.
3. **Submit code**: Check for open issues or create your own, then follow the development workflow below.

## Development Workflow
1. Fork the repository.
2. Create a new branch: `git checkout -b feature/your-feature-name`.
3. Make your changes and commit them with clear messages.
4. Push to your branch: `git push origin feature/your-feature-name`.
5. Open a Pull Request.

## Coding Standards
- Follow React best practices (Functional Components, Hooks).
- Use descriptive variable and function names.
- Ensure your code is properly commented.
- Use semi-colons and 2-space indentation.
- Follow the existing project structure.

## Pull Request Process
- Ensure your PR is linked to an issue.
- Provide a clear description of the changes made.
- Include screenshots or recordings for UI changes.
- Ensure all tests pass (if applicable).
- Wait for a maintainer to review and approve your PR.

## Code of Conduct
Be respectful and professional in all interactions. We follow the [Contributor Covenant](https://www.contributorcovenant.org/version/2/1/code_of_conduct/code_of_conduct.md).
